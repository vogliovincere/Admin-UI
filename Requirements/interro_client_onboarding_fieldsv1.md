# Interro Direct Client Onboarding — Data Collection Requirements

> **Scope:** Direct clients are entities (corporations, partnerships, LLCs, trusts, investment vehicles, etc.).  
> **Source:** Interro KYC Policy v2 (CIP, CDD, EDD sections).  
> **Purpose:** Reference for prototype application form UX and data model.

---

## Section 1 — Entity Identity (CIP)

*Minimum required for all entity clients. Maps to the Customer Identification Program.*

| Field | Notes |
|---|---|
| Legal entity name (as registered) | Exact name on formation documents |
| Doing business as (DBA) name | If applicable |
| Address of principal place of business | Street, city, state/province, country, postal code |
| U.S. EIN (Employer Identification Number) | For U.S. entities |
| Non-U.S. tax identification number | For non-U.S. entities; include issuing country |
| Government-issued certification of existence | e.g., articles of incorporation, business license, or equivalent — **file upload** |
| Certificate of Incorporation or other legal formation documents | **File upload** |

---

## Section 2 — Authorized Representative(s) (CIP)

*Individuals authorized to act on behalf of the entity. Collect for each representative.*

| Field | Notes |
|---|---|
| Full legal name | |
| Title / role at the entity | |
| Email address | For notifications and correspondence |
| Phone number | |
| Government-issued photo ID | **File upload** |

---

## Section 3 — Beneficial Owners / Control Persons (CIP)

*Required for all entity clients. UBO threshold is ≥25% at CDD; drops to ≥10% if entity is risk-rated High.*

| Field | Notes |
|---|---|
| Full legal name | Per UBO |
| Date of birth | Per UBO |
| Residential address | Per UBO |
| Ownership percentage | Per UBO |
| Role / control function (if a Control Person without ownership threshold) | e.g., CEO, Managing Partner |
| U.S. SSN or taxpayer ID | For U.S. persons |
| Non-U.S. ID number | Passport number + country of issuance, alien ID card, or other government-issued ID with photo |
| Government-issued photo ID | **File upload** per UBO |

> **UX note:** The form should support adding multiple UBO entries dynamically. The KYC policy contemplates the option of sending each UBO a separate link to self-submit their information.

---

## Section 4 — Account Purpose & Business Profile (CDD)

*Customer Due Diligence fields. Collected for all entity clients.*

| Field | Notes |
|---|---|
| Purpose of account | Free text or structured dropdown (e.g., fund transfers, treasury management, deal escrow) |
| Industry / business model | Dropdown or free text |
| Source of funds | Dropdown + description (e.g., business revenue, investor capital, proceeds from asset sales) |
| Expected transaction activity | Volume range (e.g., # of transactions/month) and value range (e.g., average transaction size) |
| Geographic exposure | Countries / jurisdictions where the entity operates or transacts |
| Ownership structure description | Narrative or org chart upload — **file upload optional** |
| Authorized Representative List | Signed list of authorized signatories — **file upload** (where applicable) |
| Proof of principal place of business address | e.g., utility bill, lease, bank statement — **file upload** (collected as needed) |

---

## Section 5 — Screening Acknowledgment (CDD)

*Collected as part of the CDD workflow; results are system-generated, not user-entered.*

| Field | Notes |
|---|---|
| Sanctions / OFAC screening consent | Checkbox / disclosure acknowledgment |
| PEP disclosure | Checkbox: does the entity have any PEP associations? (Yes / No / Unsure) |
| Adverse media acknowledgment | Disclosure that adverse media screening will be conducted |

> **UX note:** Screening results (sanctions, PEP, adverse media) are generated by Alloy and surfaced in the Interro Admin UI — not collected from the client.

## EDD — Enhanced Due Diligence (Institutional / Entity)

> **When triggered:** Client receives a composite risk score of High (2.25–3.00) in Alloy, OR Interro Compliance manually escalates.  
> **UX behavior:** EDD requests are sent to the client as a secondary request after initial onboarding. Interro Compliance can prescribe which specific items below are required for a given client.

| Field | Notes |
|---|---|
| Identification and verification of all beneficial owners ≥ 10% ownership | Full CIP data (see Section 3) for each UBO at this lower threshold — **file uploads per UBO** |
| AML program information | Description of the entity's own AML/BSA controls — **file upload or free text** |
| Corporate governance documents | Bylaws, operating agreement, partnership agreement, etc. — **file upload** |
| Organizational chart | Showing ownership and control structure — **file upload** |
| Financial statements | Most recent audited or unaudited financials — **file upload** |
| Regulatory registrations and filings | Applicable licenses, registrations, or regulatory filings — **file upload** |
| Enhanced adverse media screening | System-generated via Alloy; no client input required |
| Additional narrative on nature and purpose of account | Free-text field for Compliance-requested explanation |

> **UX note:** The EDD form should be modular — Interro Compliance selects which of the above fields/uploads to surface for each specific client request, rather than always presenting the full list.

## Field Type Summary

| Input Type | Used For |
|---|---|
| Text input | Names, addresses, IDs, descriptions |
| Dropdown / select | Industry, account purpose, source of funds categories, account type |
| Number / range | Ownership %, expected transaction volume and value |
| Date picker | Dates of birth |
| File upload | Formation docs, photo IDs, financial statements, org charts, address proof |
| Checkbox | Screening disclosures, certifications, ToS |
| Repeating group | UBO entries (supports multiple), authorized representatives |
| Auto-captured | Submission timestamp, system-generated screening results |
